<details>
<summary><b>📦 Day 01 — Variables &nbsp;&nbsp; ✅ Done</b></summary>
<br/>

| Concept | Key Rule |
|---------|----------|
| `var` | Function scoped · Hoisted as `undefined` · Can redeclare |
| `let` | Block scoped · TDZ · No redeclare · Can reassign |
| `const` | Block scoped · TDZ · No reassign · Object props mutable |
| Hoisting | Declaration moves to top — not the value |
| TDZ | `let`/`const` exist but crash if accessed before declaration |
```js
console.log(a); // undefined — var hoisted
var a = 10;

console.log(b); // ❌ ReferenceError — TDZ!
let b = 10;
```

<details>
<summary><b>📦 Day 02 — Data Types &nbsp;&nbsp; ✅ Done</b></summary>
<br/>

| Concept | Key Rule |
|---------|----------|
| 7 Primitives | String · Number · Boolean · Null · Undefined · Symbol · BigInt |
| 1 Non-Primitive | Object — Arrays and Functions are Objects |
| `typeof null` | Returns `"object"` — historic bug from 1995 |
| Primitive copy | Copied by **value** — independent copies |
| Object copy | Copied by **reference** — same memory address |
| 6 Falsy values | `false · 0 · "" · null · undefined · NaN` |
```js
let a = 10; let b = a; b = 20;
console.log(a); // 10 — unchanged (primitive)

let x = { name: "A" }; let y = x; y.name = "B";
console.log(x.name); // "B" — both changed! (reference)
```

🔗 [View Code](./Day-02/day-02-datatypes.js)

</details>

<details>
<summary><b>📦 Day 03 — Operators &nbsp;&nbsp; ✅ Done</b></summary>
<br/>

| Concept | Key Rule |
|---------|----------|
| `NaN === NaN` | Always `false` — use `isNaN()` |
| `+` operator | String on either side = concat |
| `==` vs `===` | `==` allows coercion · always use `===` |
| `??` nullish | Only triggers on `null`/`undefined` — not `0` or `""` |
| `\|\|` OR | Returns first **truthy** value |
| `&&` AND | Returns first **falsy** value |
```js
console.log(0 || "Hi");    // "Hi"
console.log(0 ?? "Hi");    // 0 ← 0 is not null/undefined
console.log([] == false);  // true — coercion trap!
```

<details>
<summary><b>📦 Day 04 — Conditionals &nbsp;&nbsp; ✅ Done</b></summary>
<br/>

| Concept | Key Rule |
|---------|----------|
| `if/else if` | First true condition wins — rest skipped |
| `switch` | Uses `===` strict comparison always |
| Fall through | Missing `break` runs all cases below |
| Early return | Exit function early — keeps code flat |
| Nested `else` | Always pairs with nearest `if` — use braces! |
```js
// Early return — clean flat code
function check(age) {
    if (age < 18) return "Minor";
    if (age > 60) return "Senior";
    return "Adult";
}
```

<details>
<summary><b>📦 Day 05 — Loops &nbsp;&nbsp; ✅ Done</b></summary>
<br/>

| Concept | Key Rule |
|---------|----------|
| `for` | Use when iterations are known |
| `while` | Use when iterations are unknown |
| `do-while` | Always runs at least once |
| `for...of` | Values — arrays, strings (not objects) |
| `for...in` | Keys — objects (keys are always strings!) |
| `break` | Exits only innermost loop |
| `continue` | Skips current iteration only |
```js
// for...in keys are STRINGS not numbers — interview trap!
let arr = ["a", "b", "c"];
for (let key in arr) {
    console.log(typeof key); // "string" ← not "number"!
}
```
